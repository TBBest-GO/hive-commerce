# 绘梦 Agent 进化路线图

**创建日期：** 2026-07-10  
**状态：** 规划中

---

## 核心进化逻辑

提示词库的积累速度决定了绘梦的专业度。最重要的是先跑通挂件品类的闭环，验证"生成→反馈→沉淀→再生成"这个循环，再复制到其他品类。

---

## 一、短期进化：技能成熟度

### 1.1 提示词库的积累与淘汰

当前 `prompt-log.md` 是记录文件，关键是建立反馈闭环：

```
生成图片 → 人工评分(1-5星) → 高分提示词沉淀 → 低分提示词淘汰/改写
```

**进化方向：**

- 每个品类积累 **10-20 个验证过的高质量模板**，覆盖主流风格
- 建立 **负面模式库**：记录哪些提示词组合效果差，避免重复踩坑
- 按 **产品 + 风格 + 场景** 三维索引，而不是简单的品类目录

**记录格式（Markdown）：**

```markdown
### YYYY-MM-DD HH:MM | 品类 | 产品名称 | 风格
- **提示词：** [完整的中文提示词]
- **负面提示词：** [中文负面提示词]
- **参数：** 模型=xxx, 分辨率=xxx, 比例=xxx
- **submit_id：** [dreamina 返回的任务 ID]
- **效果评价：** ⭐⭐⭐⭐⭐ (1-5)
- **是否沉淀：** ☐ 已沉淀到品类库 / ☐ 效果差未沉淀
- **备注：** [任何值得记录的观察]
```

### 1.2 从"通用模板"到"品类专家"

当前设计是 `通用模板兜底 → 品类库优先`，但品类库目前基本为空。

**进化路径：**

| 品类 | 深耕方向 | 达到"好用"所需迭代次数 |
|------|----------|----------------------|
| 仿真挂件 | 区分钥匙扣 vs 包挂 vs 冰箱贴的差异化表达 | ~20-30 次 |
| 服饰鞋包 | 理解面料质感、穿搭场景、季节感 | ~20-30 次 |
| 美妆护肤 | 理解成分可视化、功效表达、高级感 | ~20-30 次 |

每个品类从"能用"到"好用"大约需要 **20-30 次生成+反馈** 的迭代。

## 二、中期进化：工作流整合

### 2.1 多 Agent 协作

绘梦不是孤立的，应与其他 Agent 形成协作链：

```
策策(选题) → 绘梦(生图) → 文才(文案) → 发布
                ↑                    |
                └── 数数(数据反馈) ──┘
```

**进化方向：**

- **策策告诉绘梦"今天做什么产品"**，而不是每次人工指定
- **文才拿到图后自动配文案**，形成图文一体输出
- **数数追踪哪些图片的帖子数据好**，反馈给绘梦优化风格

### 2.2 从"接单模式"到"主动模式"

当前是 @绘梦 才工作（被动），进化后：

- 每天早上自动根据内容日历生成当日产品图
- 发现某类产品近期数据好，主动建议"要不要多做几张类似的"
- 结合节日/热点，提前准备应景的产品图

## 三、长期进化：智能化程度

### 3.1 数据驱动的提示词优化

```
小红书帖子数据(点赞/收藏/评论)
        ↓
分析哪些视觉风格表现好
        ↓
自动调整提示词权重（如"暖色调"比"冷色调"平均高30%互动）
        ↓
下一代图片自动应用
```

### 3.2 用户偏好学习

记录每个人的审美偏好：

- "老板喜欢简洁风" → 给老板出图时默认商务简约
- "运营喜欢可爱风" → 默认 kawaii
- 不同人 @绘梦 时，自动适配风格

### 3.3 跨品类迁移学习

挂件积累的"可爱风"经验，可以迁移到零食、盲盒等品类。

**进化方向：**

- 抽象出 **风格元模板**（不绑定具体品类）
- 新品类进入时，从最接近的品类模板启动，而不是从零开始

## 四、落地优先级

| 优先级 | 动作 | 预期效果 | 前置条件 |
|--------|------|----------|----------|
| 🔴 高 | 跑通 20 次挂件生成，沉淀 10 个高质量模板 | 挂件品类从"能用"到"好用" | 跑通图片生成链路 |
| 🟡 中 | 与文才 Agent 打通图文链路 | 从"单张图"到"图文内容包" | 文才 Agent 上线 |
| 🟡 中 | 接入帖子数据反馈 | 从"凭感觉"到"数据驱动" | 数数 Agent 上线 |
| 🟢 低 | 建立用户偏好档案 | 不同人 @绘梦 自动适配 | 使用一段时间后有数据 |
