# 小红书电商多智能体系统设计

**日期：** 2026-07-07  
**项目负责人：** 独立运营（1 人团队）  
**技术栈：** Hermes Agent + 飞书 + 即梦 AI (dreamina-cli)

---

## 一、项目背景

### 1.1 业务场景
- **产品品类：** 仿真面包/食物挂件（钥匙扣、包挂、冰箱贴）
- **SKU 规模：** 180+ 款（小众挂件、经典款挂件、精致挂件、好看的包挂）
- **价格区间：** 批发价 1.8 - 10.8 元/件
- **目标平台：** 小红书电商
- **目标用户：** 18-30 岁女性，喜爱可爱系/治愈系小物件

### 1.2 运营现状
- **团队：** 1 人独立运营
- **经验：** 有小红书发帖经验，无电商运营经验
- **素材：** 供应商提供产品原图
- **预算：** 5000 元（含首批进货 + AI 工具成本）
- **目标：** 短期内跑量发帖，验证是否有买家询问

### 1.3 核心诉求
通过 AI 多智能体系统，一人指挥一支 AI 团队完成电商全流程：
- **按需调用：** 在飞书 @不同机器人完成特定任务
- **自动执行：** 机器人每日自动完成固定工作（市场分析、数据追踪等）
- **即梦集成：** 自动生成图片/视频，并上传到飞书

## 二、系统架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    飞书「电商作战室」群                         │
│  @市场分析师  @内容策划师  @文案撰写师  @生图工程师              │
│  @生视频工程师  @数据分析师  @客服助理                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    Hermes Agent 集群                          │
│  每个角色 = 1 个独立 Hermes 实例                               │
│  生图/视频工程师 = 内置 dreamina-cli 技能                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    外部工具集成                                │
│  dreamina-cli (自动生图/生视频)  小红书 API  飞书 Bot API      │
│  ↑ 生成的图片/视频 → 通过飞书 Bot 直接发到群里                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术选型理由

**选择 Hermes Agent 而非 OpenClaw：**
- ✅ **原生飞书集成：** 扫码即可接入，有中文教程和部署文档
- ✅ **自学习循环：** 从经验中创建技能并持续改进，适合长期运营
- ✅ **持久记忆：** 跨会话记住项目上下文，适合电商数据积累
- ✅ **MIT 开源：** 无追踪，可自托管，成本可控

## 三、Agent 角色定义

### 3.1 市场分析师

**核心职责：** 抓取小红书热门话题、竞品动态、品类趋势

**输入：**
- 品类关键词（仿真面包挂件、可爱钥匙扣、治愈系小物等）
- 竞品账号列表（初始列表由你提供，后续 Agent 自动发现新竞品）

**输出：**
- 每日热点报告（飞书文档）
- 选品建议（哪些款式可能爆）
- 热门话题标签推荐
- 竞品账号发现报告（自动发现同品类新账号）

**自动任务：**
- 每日 9:00 输出"今日热点简报"
- 每周一 10:00 输出"上周竞品分析周报"

**数据获取方式：**
- **小红书数据：** 通过网页爬虫抓取（小红书无官方开放 API），使用 Selenium/Playwright 模拟浏览器访问
- **飞书文档存储：** 抓取的数据存储到飞书多维表格，便于后续分析
- **注意事项：** 爬虫需遵守小红书 robots.txt，控制抓取频率（每分钟不超过 10 次）

**Hermes 技能配置：**
```yaml
skills:
  - xhs-trend-scraper: 抓取小红书热门话题（网页爬虫）
  - competitor-tracker: 追踪竞品账号动态（定时抓取竞品主页）
  - trend-analyzer: 分析品类趋势（LLM 分析抓取的数据）
  - competitor-discoverer: 自动发现同品类新账号
```

### 3.4 生图工程师

**核心职责：** 调用即梦 CLI 自动生成图片，上传到飞书

**输入：**
- 供应商原图
- 文案撰写师的文案简报
- 风格要求（可爱风、生活场景、特写展示等）

**输出：**
- 生成的图片文件（直接上传到飞书群）
- 即梦提示词记录（便于复盘优化）

**工作流：**
```
1. 接收供应商原图 + 文案简报
2. 生成 3 组不同风格的即梦提示词：
   - 可爱风：突出产品的萌感，背景用柔和色调
   - 生活场景：产品挂在包上/放在桌上的场景
   - 特写展示：细节纹理展示
3. 调用 dreamina-cli:
   - dreamina generate --prompt "..." --input supplier_img.jpg --style cute
   - dreamina generate --prompt "..." --input supplier_img.jpg --style lifestyle
   - dreamina generate --prompt "..." --input supplier_img.jpg --style closeup
4. 下载生成的图片
5. 通过飞书 Bot API 上传图片到群聊
6. 记录提示词和生成结果到飞书文档
```

**即梦提示词模板示例：**
```
风格：可爱风
提示词：cute simulated bread keychain, kawaii style, soft pastel background, 
        studio lighting, high detail, 4k, product photography
负面提示词：blurry, low quality, distorted

风格：生活场景
提示词：simulated bread keychain hanging on bag, lifestyle photography, 
        coffee shop background, natural lighting, cozy atmosphere
负面提示词：unrealistic, overexposed

风格：特写展示
提示词：close-up shot of simulated bread keychain, detailed texture, 
        macro photography, studio lighting, sharp focus
负面提示词：blurry, noisy
```

**Hermes 技能配置：**
```yaml
skills:
  - dreamina-cli-integration: 调用即梦 CLI 生成图片
  - prompt-engineer: 生成即梦提示词
  - feishu-uploader: 上传图片到飞书
  - prompt-logger: 记录提示词和生成结果
```

### 3.5 生视频工程师

**核心职责：** 调用即梦 CLI 自动生成视频，上传到飞书

**输入：**
- 产品图（供应商图或生图工程师生成的图）
- 文案撰写师的文案简报
- 视频风格要求（展示类、教程类、氛围类等）

**输出：**
- 生成的视频文件（直接上传到飞书群）
- 即梦视频提示词记录

**工作流（三层漏斗式，参考 jimeng/workflow.md）：**
```
第 1 层：试片验证（低成本）
  - dreamina video --duration 5s --resolution 720p --model fast --prompt "..."
  - 上传试片到飞书，等待快速确认方向
  - 成本：约完整成片的 1/6

第 2 层：成片升级（确认后）
  - dreamina video --duration 10s --resolution 1080p --model vip --prompt "..."
  - 上传成片到飞书

第 3 层：多镜头扩展（可选）
  - 生成 2-3 个角度分段 → 剪辑拼接
  - 全景 → 特写 → 氛围
```

**即梦视频提示词模板示例：**
```
试片提示词（5s, 720p）:
simulated bread keychain rotating slowly, soft lighting, 
smooth camera movement, product showcase

成片提示词（10s, 1080p）:
simulated bread keychain hanging on bag, walking in coffee shop,
natural lighting, cozy atmosphere, lifestyle vlog style

运镜提示词:
camera pan from left to right, slow zoom in, smooth tracking shot
```

**Hermes 技能配置：**
```yaml
skills:
  - dreamina-video-cli: 调用即梦 CLI 生成视频
  - video-prompt-engineer: 生成即梦视频提示词
  - three-layer-funnel: 三层漏斗式视频制作流程
  - feishu-video-uploader: 上传视频到飞书
```

### 3.6 数据分析师

**核心职责：** 追踪帖子表现，输出优化建议

**输入：**
- 小红书帖子数据（通过网页爬虫抓取，和市场分析师共用爬虫基础设施）
- 发布时间、内容类型、话题标签
- 你的手动标注（哪些帖子带来了询盘/成交）

**输出：**
- 每日数据日报（飞书文档）
- 优化建议（什么类型的内容表现好、最佳发布时间等）
- 爆款复盘（分析高表现帖子的共同特征）
- 询盘转化追踪（哪些内容带来了实际询问）

**数据获取方式：**
- **帖子数据：** 通过网页爬虫抓取自己账号的主页，获取每条帖子的阅读、点赞、收藏、评论数
- **询盘数据：** 你手动记录（每天花 2 分钟在飞书表格里记录：今日收到几个询盘、来自哪条帖子）
- **注意事项：** 只抓取自己账号的数据，不抓取他人数据，遵守平台规则

**自动任务：**
- 每日 22:00 输出"今日数据日报"
- 每周五 18:00 输出"本周数据周报 + 优化建议"

**数据日报模板：**

```python
    # 上传视频（示例）
    elif file_path.endswith(('.mp4', '.mov')):
        file_key = upload_video(file_path)  # 需要先实现文件上传接口
        body = {
            "receive_id": chat_id,
            "msg_type": "media",
            "content": f'{{"file_key": "{file_key}"}}'
        }
    
    requests.post(url, headers=headers, json=body)
```

**实际开发步骤：**
1. 在[飞书开放平台](https://open.feishu.cn/)创建应用，获取 app_id 和 app_secret
2. 申请权限：`im:message:send`（发送消息）、`im:file:upload`（上传文件）
3. 参考[飞书 Bot 文档](https://open.feishu.cn/document/home/develop-a-bot-in-5-minutes)实现文件上传
4. 测试图片/视频消息发送

### 8.2 首批进货成本

**原始方案（超出预算）：**
- 选择 10-15 个 SKU 试水
- 每个 SKU 进货 300 件（最低起订量）
- 平均单价 5 元
- **总计：10 × 300 × 5 = 15,000 元** ❌ 超出 5000 预算

**推荐方案（符合预算）：**
- 选择 **3 个 SKU** 试水（建议：小汉堡、冰淇淋、牛角包——最受欢迎的 3 类）
- 每个 SKU 进货 300 件（最低起订量）
- 平均单价 5 元
- **总计：3 × 300 × 5 = 4,500 元** ✅

**供应商协商建议：**
1. 尝试协商小批量试货（100 件/SKU），可降低到 1,500 元
2. 或与供应商说明是电商测试，争取首单优惠
3. 如协商成功，可多选 2-3 个 SKU

### 8.3 总预算分配（推荐方案）

| 项目 | 预算 | 说明 |
|------|------|------|

## 九、风险与应对

### 9.1 技术风险

**风险：** 即梦 CLI 生成质量不稳定  
**应对：** 采用三层漏斗式流程，先试片验证再成片，降低失败成本

**风险：** 飞书 Bot API 调用限制  
**应对：** 批量上传、缓存 token、监控 API 调用频率

**风险：** Hermes Agent 部署复杂  
**应对：** 参考官方文档和中文教程，逐步部署

### 9.2 业务风险

**风险：** 品类不受欢迎，没有买家询问  
**应对：** 小成本试错（3 SKU），快速验证，不行就换品类

**风险：** 内容同质化，无法吸引用户  
**应对：** 数据分析师追踪表现，持续优化内容策略

**风险：** 即梦生成图片/视频与实物不符  
**应对：** 结合供应商原图 + AI 生成，保持真实性
